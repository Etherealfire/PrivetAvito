## BUG-01 Повторное удаление объявления возвращает 500

**Endpoint:** DELETE /api/2/item/{id}

**Шаги воспроизведения:**
1. Создать объявление
2. Удалить объявление
3. Повторно удалить то же объявление

**Ожидаемый результат:**
- HTTP 404 Not Found

**Фактический результат:**
- HTTP 500 Server Error

## BUG-02 Получение объявлений по несуществующему sellerID

**Endpoint:** GET /api/1/{sellerID}/item

**Шаги воспроизведения:**
1. Отправить запрос с несуществующим sellerID

**Ожидаемый результат:**
- HTTP 404 Not Found или 400 Bad Request

**Фактический результат:**
- HTTP 200 OK
- Пустой массив в ответе

---

## BUG-03 Отрицательный sellerID принимается системой

**Endpoint:** GET /api/1/{sellerID}/item

**Шаги воспроизведения:**
1. Передать отрицательный sellerID (например, -111121)

**Ожидаемый результат:**
- HTTP 400 Bad Request

**Фактический результат:**
- HTTP 200 OK
- Пустой ответ

## BUG-04 Некорректные сообщения об ошибках при валидации данных

**Описание:**
При передаче невалидных данных API возвращает сообщения об ошибках, не соответствующие реальной причине ошибки.

**Примеры:**
- Пустое поле `name` → сообщение "не передан объект - объявление"
- Некорректный JSON → сообщение "не передан объект - объявление"
- `price = 0` → сообщение "поле price обязательно"

**Ожидаемо:**
Сообщения должны явно указывать причину ошибки.
